# Improved Prompt: Robust API Key Management & Model Selection

## Objective
Fix the API Key "Delete" and "Test" workflows to be fully reactive without page reloads, and introduce a Model Selection setting with descriptive options.

## 1. Fix: Reactive API Key Deletion
**Issue**: The delete button currently fails to update the UI state effectively, leading to stale data during testing.

**Requirements**:
- **Immediate Feedback**: When "Delete Key" is clicked:
  1.  **Frontend**: Instantly clear the API Key input field (`val('')`).
  2.  **Backend**: Trigger `air_delete_api_key` AJAX to remove the key from the database.
- **Live Testing**: The "Test Connection" logic must NOT rely on the saved database value if the input field is empty.
  - If the input is cleared (via Delete or manual backspace), clicking "Test Connection" must send the empty value to the server.
  - The server should respond with "No API Key provided" (or similar error) rather than falling back to the saved key.

## 2. Feature: AI Model Selection
**Objective**: Allow users to switch between specific Llama 4 models.

**Requirements**:
- **Settings Field**: Add a new "AI Model" section/field (Radio Button group recommended for visibility).
- **Options**:
  1.  **Maverick** (`meta-llama/llama-4-maverick-17b-128e-instruct`)
      - *Description*: "High-performance model for detailed and creative image analysis."
  2.  **Scout** (`meta-llama/llama-4-scout-17b-16e-instruct`)
      - *Description*: "Lightweight model optimized for speed and efficiency."
- **Persistence**: Save the selection to the `air_options` array.
- **Integration**: Update `Groq_Service` to use the selected model ID dynamically (falling back to Maverick as default).

## Success Criteria
- [ ] Clicking "Delete Key" clears the input AND deletes the DB entry.
- [ ] Clicking "Test Connection" immediately after deleting (without reload) returns an error (proving the key is gone/empty).
- [ ] Changing the Model selection saves correctly.
- [ ] Image renaming uses the newly selected model ID.
